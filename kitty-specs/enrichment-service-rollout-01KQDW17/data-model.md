# Enrichment Service Rollout Data Model

## EnrichmentRequest

- `lead_id`: required string identifier supplied by Waaseyaa.
- `company_name`: required string used by all enrichment types.
- `domain`: optional string used by company and tech-stack enrichers.
- `sector`: optional string used as search context.
- `requested_types`: required non-empty list of enrichment types. Supported values are `company_intel`, `tech_stack`, and `hiring`.
- `signals`: optional request-local signal map supplied by Waaseyaa.
- `callback_url`: required URL used for asynchronous result delivery.
- `callback_api_key`: required secret used only for the outbound `X-Api-Key` callback header.

## EnrichmentResult

- `lead_id`: original lead identifier.
- `type`: enrichment type that produced the result.
- `status`: result status such as `success`, `empty`, or `error`.
- `confidence`: deterministic numeric confidence score.
- `data`: enrichment-specific JSON object.
- `error`: optional error message for failed enrichment attempts.

## CallbackEnvelope

- `lead_id`: original lead identifier.
- `results`: list of enrichment results.
- `completed_at`: timestamp generated by the enrichment service.

## Enricher

- `Type() string`: returns the enrichment type key.
- `Enrich(ctx, request) (EnrichmentResult, error)`: runs the requested lookup and returns a result or an error that can be isolated from other requested types.

## Relationships

- One `EnrichmentRequest` can request many enrichment types.
- Each known requested type maps to one registered `Enricher`.
- Each `Enricher` produces one `EnrichmentResult`.
- The callback client sends one callback payload to the request-provided `callback_url`.
