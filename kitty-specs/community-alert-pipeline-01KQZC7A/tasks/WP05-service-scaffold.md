# WP05 — Service Scaffold

## Objective

Stand up the `alert-crawler/` directory inside the north-cloud monorepo with the operational scaffolding mirroring `signal-crawler`'s pattern: oneshot Go binary, multi-stage Alpine Dockerfile, per-service Taskfile, config.yml with intentional blank fields where SetDefaults applies, `.layers` 3-tier model, and a CLAUDE.md skeleton. Includes the `replace` directive in `go.mod` so Phase B can develop in parallel with Phase A.

This WP is the foundation: every subsequent Phase B WP depends on this layout.

### T016 — Create `alert-crawler/` directory and `go.mod`

**Purpose**: Establish the Go module with the `replace` directive that bridges Phase A and Phase B.

**Steps**:
1. Create directory `/home/jones/dev/north-cloud/alert-crawler/`.
2. Initialize the module:
   ```bash
   cd alert-crawler
   go mod init github.com/jonesrussell/north-cloud/alert-crawler
   ```
3. Add the temporary `replace` directive in `go.mod`:
   ```
   replace github.com/jonesrussell/indigenous-taxonomy => ../../indigenous-taxonomy
   ```
   (Path is relative to the `alert-crawler` directory.)
4. Add `require github.com/jonesrussell/indigenous-taxonomy v0.0.0` placeholder; the `replace` directive points it at the local path. WP19 will pin the released version.
5. Add other immediate dependencies: `github.com/mattn/go-sqlite3`, plus indirect via `infrastructure/*` packages from north-cloud root (`go.work` resolves these in IDE; runtime build path uses local module replace where applicable).
6. Run `go mod tidy` to finalize.

**Files**:
- `alert-crawler/go.mod` (new)
- `alert-crawler/go.sum` (new, generated by go mod tidy)

**Validation**:
- `cd alert-crawler && go build ./...` succeeds (even with empty internal packages).
- `replace` directive correctly resolves to `../indigenous-taxonomy` (relative to the alert-crawler module root).

**Edge Cases**:
- The north-cloud monorepo uses `GOWORK=off` per service per CLAUDE.md. Make sure module-resolution does not lean on the workspace.

### T018 — Create `config.yml`

**Purpose**: Default configuration file shipped inside the Docker image. Critical: leave fields blank where `SetDefaults` provides the actual default value (RR-007 pitfall).

**Steps**:
1. Create `alert-crawler/config.yml`:
   ```yaml
   # Alert-crawler default configuration.
   #
   # IMPORTANT: Fields owned by config.SetDefaults must be left BLANK here.
   # Non-empty YAML values silently override SetDefaults (see CLAUDE.md gotcha
   # and RR-007). Only set fields you need to override per-environment.

   # service:
   #   name: ""  # SetDefaults

   # sources:
   #   - id: "mhrn"
   #     name: "Manitoba Harm Reduction Network"
   #     feed_url: ""             # SetDefaults
   #     acquisition_strategy: ""  # SetDefaults  (default: rss)
   #     poll_interval: ""         # SetDefaults  (default: 30m)
   #     default_category: ""      # SetDefaults  (default: harm_reduction)
   #     default_scope: []         # SetDefaults  (default: [treaty:1, canada:manitoba])
   #     default_expiry: ""        # SetDefaults  (default: 720h)
   #     enabled: true

   # severity:
   #   table:
   #     carfentanil: critical
   #     nitazenes: high
   #     # SetDefaults populates this if empty
   ```
2. The intentional all-comment shape signals "all fields default-controlled". Operators uncomment specifically what they need to override.

**Files**:
- `alert-crawler/config.yml` (new, ~25 lines, mostly comments)

**Validation**:

### T020 — Create multi-stage `Dockerfile`

**Purpose**: Match signal-crawler's container shape (Alpine multi-stage, non-root uid 1000, CGO enabled for SQLite, `config.yml` baked in).

**Steps**:
1. Read `/home/jones/dev/north-cloud/signal-crawler/Dockerfile` for the canonical pattern.
2. Create `alert-crawler/Dockerfile` matching the structure:
   ```dockerfile
   # syntax=docker/dockerfile:1
   FROM golang:1.26.2-alpine AS builder
   RUN apk add --no-cache gcc musl-dev
   WORKDIR /build
   COPY ../indigenous-taxonomy /indigenous-taxonomy   # for replace directive
   COPY alert-crawler/go.mod alert-crawler/go.sum ./
   RUN go mod download
   COPY alert-crawler/ .
   ENV CGO_ENABLED=1
   RUN go build -o /alert-crawler .

   FROM alpine:3.21
   RUN addgroup -g 1000 alert-crawler && adduser -u 1000 -G alert-crawler -D -s /sbin/nologin alert-crawler
   COPY --from=builder /alert-crawler /alert-crawler
   COPY alert-crawler/config.yml /etc/alert-crawler/config.yml
   USER alert-crawler
   WORKDIR /app
   VOLUME ["/app/data"]
   ENTRYPOINT ["/alert-crawler"]
   ```
3. The `COPY ../indigenous-taxonomy` line bakes the sibling repo into the image during Phase B (while `replace` is active). Removed in WP19.
4. Verify `docker build -f alert-crawler/Dockerfile -t alert-crawler:dev .` succeeds from the north-cloud root.

**Files**:
- `alert-crawler/Dockerfile` (new, ~30 lines).

**Validation**:
- Image builds.
- `docker run --rm alert-crawler:dev` exits 0 (since main.go is a stub).
- Image runs as uid 1000.

**Edge Cases**:
- Build context must include `../indigenous-taxonomy/` for the replace directive. Document this in WP05's CLAUDE.md.
